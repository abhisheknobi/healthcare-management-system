Core Architecture & Technology Stack: The project is structured as a decoupled application featuring a high-performance RESTful API backend built with Java 21 and Spring Boot 4.0.0. The backend manages the core business logic while communicating over HTTP (via Axios) with the frontend to deliver a seamless user experience.
Advanced User Management & Security (AAA Principals): Implements strict Role-Based Access Control (RBAC) supporting distinct roles (ADMIN, DOCTOR, PATIENT, RECEPTIONIST) utilizing Spring Security 6.x. It enforces stateless, token-based authentication via JWT (JSON Web Tokens) to secure all REST endpoints. Credentials are automatically hashed and protected using BCryptPasswordEncoder, alongside robust CORS configuration and CSRF protection.
Patient & Clinical Data Layer (EHR & Prescriptions): Utilizes Spring Data JPA interfaced with PostgreSQL to map and persist complex relational models (e.g., One-To-Many for Doctor-Appointments, Many-To-Many for Patient-Medications). Functions as a comprehensive Electronic Health Record (EHR) system that tracks medical history, allergies, and chronic conditions, while housing a secure prescription engine that allows doctors to dynamically generate and link digital prescriptions to specific appointments.
Highly-Concurrent Appointment Workflow: Implements a real-time scheduling module that manages doctor availability slots and dynamically tracks appointment lifecycles (Pending, Confirmed, Completed, Cancelled). The backend handles multiple simultaneous booking requests with strict concurrency control and transaction management algorithms to completely prevent race conditions and slot double-booking.
Billing, Pharmacy Integration & System Auditability: Features an automated invoicing engine that calculates bills based on dynamic consultation fees and laboratory tests, fully integrated with a pharmacy tracker for dispensed medications. Furthermore, the system is strictly validated at the controller level using Hibernate Validator (e.g., @NotNull, @Email, @Size) and maintains rigorous auditability by logging every critical state change with a precise timestamp and the corresponding user ID.
